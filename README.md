# Student Group Curator Management System

Консольное приложение для управления информацией о студентах, группах и кураторах.  
Реализовано на C# с использованием **Entity Framework Core** и **PostgreSQL**.  
Проект выполнен в рамках лабораторной работы и демонстрирует многослойную архитектуру, работу с базой данных и LINQ-запросы.

## Описание задачи

Необходимо реализовать систему, позволяющую хранить и обрабатывать данные о студентах, учебных группах и кураторах.  
Связи между сущностями:

- **Студент (Student)** может состоять только в одной группе.
- **Группа (Group)** может содержать произвольное количество студентов и имеет одного куратора.
- **Куратор (Curator)** может быть куратором только у одной группы.

Помимо стандартных CRUD-операций, программа выполняет три аналитических запроса по требованию пользователя:

1. Подсчёт количества студентов в группе по её идентификатору.
2. Вывод имени куратора группы, в которой учится студент с заданным ID.
3. Расчёт среднего возраста студентов в группе, курируемой куратором с заданным ID.

## Технологии и инструменты

- **Язык:** C# (.NET 6/8)
- **ORM:** Entity Framework Core 6/8
- **СУБД:** PostgreSQL (локальная установка)
- **Среда разработки:** Visual Studio 2022 / Rider
- **Система контроля версий:** Git

## Структура решения

Решение состоит из четырёх проектов, разделённых по слоям:

| Проект              | Тип                  | Назначение                                                                 |
|---------------------|----------------------|----------------------------------------------------------------------------|
| `Presentation`      | Console Application  | Пользовательский интерфейс (меню, ввод/вывод). Взаимодействует с фасадом. |
| `Core`              | Class Library        | Интерфейсы, фасад, бизнес-логика, доменные модели (DTO/конвертеры).       |
| `DatabaseModels`    | Class Library        | Модели базы данных (сущности `Student`, `Group`, `Curator`).              |
| `DatabaseContext`   | Class Library        | Контекст БД (`DbContext`) и миграции Entity Framework.                    |

**Принципы разработки:**  
- Чёткое разделение ответственности (слои ввода/вывода, логики, данных).  
- Использование DTO и конвертеров для отделения моделей БД от внутреннего представления.  
- Обработка ошибок ввода и отсутствия данных (без падений приложения).  
- Следование принципам KISS, DRY, YAGNI.

## Подготовка и запуск

### Требования
- Установленная **PostgreSQL** (локально).
- **.NET SDK** (версии 6.0 или выше).
- Среда разработки (Visual Studio / Rider) или консоль.
